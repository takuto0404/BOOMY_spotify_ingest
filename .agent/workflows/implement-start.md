# 実装プロセス

$ARGUMENTS で指定された仕様書から実装を開始します。
TodoWriteツールを使ってフェーズごとにTODOを作成してください。

## フェーズ1: 
**このフェーズではsubagentの使用は禁止です**
仕様書がフォルダで渡された場合、必ず0番目のファイルから全て読んで、完成した仕様書と、それに至るまでの過程を理解してください。 

## フェーズ2: 
プロジェクトのコーディング規約や、アーキテクチャを把握します。

## フェーズ3: 
1. 仕様書に基づき実装タスクを分解してTodoWriteツールを使いTODOを更新してください。
2. 仕様書に基づき実装を開始してください

## フェーズ4:
仕様書通りに実装が完了しているか、ステップバイステップで確認してください

## フェーズ5:
仕様書通りに実装で動作可能か検証してください
- コンパイルエラーの検証
- linterの検証
- 指示がある場合はgitのコミット

## フェーズ6:
仕様書通りの実装が完了したかユーザーにレポートしてください
