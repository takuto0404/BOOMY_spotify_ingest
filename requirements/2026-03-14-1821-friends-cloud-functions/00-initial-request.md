# Initial Request

このドキュメントは Cloud Functions 実装担当にそのまま渡すことを目的とする。 Firestoreは公開読み取り。更新はすべて Cloud Functions 経由で行う。 - 認証済みユーザーのみが呼び出し可能（Firebase Auth）。 - `context.auth.uid` を呼び出しユーザーIDとして使用する。 - 申請拒否は履歴を残さず削除する（`rejected` は使わない）。 - クライアントは `rejected` を扱わない。 - `users/{userId}/profile` - `userId` (string) - `userName` (string, unique予定) - `displayName` (string) - `createdAt` (timestamp) - `updatedAt` (timestamp) - `users/{userId}/friends/{friendUserId}` - `friendUserId` (string) - `createdAt` (timestamp) - `friendRequests/{requestId}` - `fromUserId` (string) - `toUserId` (string) - `status` (string: `pending` / `accepted` / `canceled`) - `createdAt` (timestamp) - `updatedAt` (timestamp) - `sendFriendRequest` - `acceptFriendRequest` - `cancelFriendRequest` - `rejectFriendRequest` - `removeFriend` - `getUserProfileById` **入力** - `toUserId` (string) **出力** - `requestId` (string) - `status` (string: `pending`) **機能条件** - `toUserId` が存在しない場合は `not_found`。 - すでにフレンドの場合は `already_friends`。 - 既存の `pending` リクエストが存在する場合は `request_exists`。 - 自分自身への申請は `invalid_state`。 - 成功時、`friendRequests` に `pending` を作成。 **入力** - `requestId` (string) **出力** - `friendUserId` (string) - `status` (string: `accepted`) **機能条件** - `requestId` が存在しない場合は `not_found`。 - `toUserId` が `context.auth.uid` と一致しない場合は `permission_denied`。 - `status != pending` は `invalid_state`。 - 成功時: - `friendRequests` を `accepted` に更新 - 双方向に `users/{userId}/friends/{friendUserId}` を作成 **入力** - `requestId` (string) **出力** - `status` (string: `canceled`) **機能条件** - `requestId` が存在しない場合は `not_found`。 - `fromUserId` が `context.auth.uid` と一致しない場合は `permission_denied`。 - `status != pending` は `invalid_state`。 - 成功時: - `friendRequests` を `canceled` に更新 **入力** - `requestId` (string) **出力** - `status` (string: `canceled`) **機能条件** - `requestId` が存在しない場合は `not_found`。 - `toUserId` が `context.auth.uid` と一致しない場合は `permission_denied`。 - `status != pending` は `invalid_state`。 - 成功時: - `friendRequests` を削除（履歴を残さない） **入力** - `friendUserId` (string) **出力** - `status` (string: `canceled`) **機能条件** - `friendUserId` が存在しない場合は `not_found`。 - 成功時: - 双方向の `users/{userId}/friends/{friendUserId}` を削除 - 既存の `friendRequests` があれば `canceled` に更新して整合性維持 **入力** - `userId` (string) **出力** - `userId` (string) - `userName` (string) - `displayName` (string) **機能条件** - `userId` が存在しない場合は `not_found`。 - 自分のプロフィール取得も許可。 - `not_found` - `already_friends` - `request_exists` - `invalid_state` - `permission_denied` - `userName` は将来的に検索対象。現段階はID検索のみ。 - 申請拒否は履歴を残さず削除する。 - Firestoreは公開読み取りだが、書き込みはCloud Functionsのみ。
